# 위키 운영 절차

> Ingest / Query / Lint 각 Operation의 단계별 실행 규칙.

---

## Ingest — 소스 → 위키 통합

**새로운 지식이 생겼을 때 위키에 반영하라.**

### 트리거 시점

| 시점 | 예시 |
|------|------|
| 도메인 개발 완료 후 | 새 화면의 SP 목록, 화면 구조, 특이사항 |
| 코드 탐색 중 미문서화 패턴 발견 시 | 팝업→그리드 연동 패턴 |
| 세션 답변이 재사용 가치 있을 때 | SP 디버깅 순서, 파라미터 불일치 사례 |
| 세션 종료 시 미등록 도메인 감지 시 | stop 훅이 `/wiki-ingest` 제안 |

### 절차

1. `index.md`에서 대상 개념 존재 여부 확인
2. **존재 시**: 기존 노트 업데이트, `last_updated` 갱신
3. **없을 시**: 분류 결정 → 신규 노트 생성 (`@rules/templates.md` 참고)
4. `index.md` 해당 섹션 테이블에 항목 추가 또는 날짜 갱신
5. `log.md` 최상단에 항목 prepend
6. 기존 관련 노트에 역방향 `[[wikilink]]` 추가

### 분류 결정 규칙

- 코드 파일(Controller/Service 등) → `domain/`
- 반복 구현 방법, 해결책 → `pattern/`
- 용어, 규칙, 정의 → `concept/`
- 판단 불가 시 → `domain/`으로 시작, 나중에 분리

## Query — 위키 기반 답변

**코드 탐색 전 반드시 위키를 먼저 확인하라.**

### 절차

1. `index.md` 읽기 → 키워드 매칭으로 관련 페이지 식별 (최대 5개)
2. 관련 페이지 읽기 → `## 연결` 섹션으로 추가 탐색
3. 위키 내용으로 답변 가능 → 즉시 답변, 출처 명시
4. 위키 내용 불충분 → 소스 탐색 후 `/wiki-ingest` 제안

### 답변 출처 명시 규칙

답변 시 반드시 출처를 밝혀라:

```
출처: [[.wiki/concept/base-dto]], [[.wiki/pattern/cud-service]]
```

### Filing 규칙

답변이 위키에 없던 내용을 포함했다면 → `/wiki-ingest`로 summary 노트 생성을 제안하라.

## Lint — 위키 건강 점검

**정기적으로 또는 대규모 변경 후 실행하라. (권장: 월 1회)**

### 점검 항목

| 항목 | 정의 | 감지 방법 |
|------|------|----------|
| 고아 노트 | `.wiki/`에 있으나 `index.md`에 없는 노트 | Glob + index.md 비교 |
| 깨진 링크 | 존재하지 않는 파일을 참조하는 `[[wikilink]]` | 링크 추출 + 파일 존재 확인 |
| 고립 노트 | `## 연결` 섹션 링크 0개인 노트 | 섹션 파싱 |
| 오래된 노트 (`--deep`) | 30일 이상 위키 갱신 없는데 관련 코드 변경됨 | git log + last_updated 비교 |

### --fix 수정 범위

자동 수정 가능:
- 연결 없는 노트: `## 연결` 섹션에 관련 링크 추가

수동 처리 필요:
- 고아 노트: 내용 검토 후 index.md 등록 또는 삭제
- 깨진 링크: 파일명 수정 또는 링크 제거

---

## index.md 갱신 규칙

**Ingest 완료 후 반드시 index.md를 갱신하라.**

```markdown
## Concept
| 페이지 | 요약 | 갱신 |
|--------|------|------|
| [[.wiki/concept/base-dto]] | 공통 DTO 기반 클래스, errorCode 필드 | 2026-05-04 |

## Pattern
| 페이지 | 요약 | 갱신 |
|--------|------|------|

## Domain
| 페이지 | 요약 | 갱신 |
|--------|------|------|
```

- 행 형식: `| [[.wiki/분류/파일명]] | 한줄 요약 | YYYY-MM-DD |`
- `last_updated` frontmatter도 오늘 날짜로 갱신

---

## log.md 갱신 규칙

**모든 Ingest/Query/Lint 후 log.md 최상단에 항목을 추가하라.**

```markdown
## [YYYY-MM-DD] ingest | {노트 제목}
- **출처**: {소스 경로 또는 맥락}
- **생성**: [[.wiki/xxx/yyy]]
- **갱신**: [[.wiki/xxx/zzz]] (역링크 추가) — 있을 때만 포함
- **index.md**: 카탈로그 항목 추가

## [YYYY-MM-DD] query | {질문 제목}
- **질문**: {무엇을 물어봤는지}
- **결과**: filed: [[링크]] / not filed: {이유}

## [YYYY-MM-DD] lint | {점검 범위}
- **발견**: 고아 {N}개, 깨진 링크 {N}개
- **조치**: {수행한 수정}
```

❌ **금지**: log.md 하단에 추가 (항상 최상단 prepend)
